moderate Regular Expression Denial of Service in postcss
  • Module: postcss
  • Published: March 11th 2022
  • Reported by:
  • CWE-400
  • CVE-2021-23382
  • Vulnerable: <7.0.36
  • Patched: >=7.0.36
  • Exploitability:

Overview

The package postcss versions before 7.0.36 or between 8.0.0 and 8.2.13 are vulnerable to Regular Expression Denial of Service (ReDoS) via getAnnotationURL() and loadAnnotation() in lib/previous-map.js. The vulnerable regexes are caused mainly by the sub-pattern /*\s* sourceMappingURL=(.*).

Findings

  • react-scripts>resolve-url-loader>postcss

Remediation

Upgrade to version 7.0.36 or later

References

moderate Improper Neutralization of Special Elements used in an OS Command.
  • Module: react-dev-utils
  • Published: March 11th 2022
  • Reported by:
  • CWE-78
  • CVE-2021-24033
  • Vulnerable: >=0.4.0 <11.0.4
  • Patched: >=11.0.4
  • Exploitability:

Overview

react-dev-utils prior to v11.0.4 exposes a function, getProcessForPort, where an input argument is concatenated into a command string to be executed. This function is typically used from react-scripts (in Create React App projects), where the usage is safe. Only when this function is manually invoked with user-provided values (ie: by custom code) is there the potential for command injection. If you're consuming it from react-scripts then this issue does not affect you.

Findings

  • react-scripts>react-dev-utils

Remediation

Upgrade to version 11.0.4 or later

References

  • https://nvd.nist.gov/vuln/detail/CVE-2021-24033
  • https://github.com/facebook/create-react-app/pull/10644
  • https://github.com/facebook/create-react-app/commit/f5e415f3a5b66f07dcc60aba1b445fa7cda97268
  • https://www.facebook.com/security/advisories/cve-2021-24033
  • https://github.com/advisories/GHSA-5q6m-3h65-w53x
